Principes de design

  • Clarté avant tout — chaque donnée doit être lisible au premier coup d'œil
  • Contraste fort — particulièrement en dark mode, pour des environnements d'opération intensive
  • Cohérence marque — les couleurs Flow sont directement héritées du logo eLogik
  • Accessibilité — toutes les combinaisons de couleurs respectent WCAG AA minimum

🔤 Typographie

Police principale : Plus Jakarta Sans

Choisie pour sa personnalité moderne sans être froide, et son excellente lisibilité aux petites tailles dans les tableaux de données.

Source : Google Fonts — Plus Jakarta Sans

Police mono : DM Mono — pour les codes commandes, références, données techniques

Échelle typographique

TokenTaillePoidsUsage

--text-mono12px400 RegularCodes, références, données brutes

Règles typographiques

  • Toujours sentence case — jamais ALL CAPS dans les titres principaux
  • Les grands chiffres (KPIs) utilisent weight 300 pour paraître légers et lisibles
  • Les labels de colonnes sont en uppercase + letter-spacing 0.08em
  • DM Mono est réservé aux données identifiantes : CMD-48291, 1Z999AA…

🧩 Molecules — Composants composites

Search Bar

Combinaison : icône search + input + raccourci clavier.

  • Raccourci global : ⌘K affiché à droite
  • Largeur recommandée : 280–360px
  • Toujours accessible depuis la Top Bar

Barre de filtres

Combinaison : Search Bar + Tags de catégorie + bouton "Filtres ▾" (filtres avancés).

  • Les tags actifs affichent un ✕ pour suppression rapide
  • Maximum 5 tags visibles — au-delà, afficher +N autres

Metric Card

Structure : label uppercase (10px) → chiffre principal (28px 300) → delta badge + contexte.

ÉlémentRègle
Chiffre principalToujours weight 300, letter-spacing −0.03em
Delta positifBadge vert lime background
Delta négatifBadge rouge background
Progress barOptionnelle, hauteur 4px, border-radius pill

Pagination

  • Toujours afficher : ‹ prev + pages numérotées + next ›
  • Page active : background cyan primary, texte blanc
  • Ellipsis … si plus de 7 pages
  • Afficher le nombre total de résultats au-dessus (1–25 sur 248 résultats)

Table Row

🏛️ Organisms — Patterns complets

Data Table

Structure complète :

  1. Toolbar : titre + search bar + filtres + export
  2. En-têtes : uppercase 10px, sortable (icône ↕), sticky au scroll
  3. Lignes : hover state + sélection checkbox optionnelle
  4. Footer : pagination + nombre de résultats

Règles :

✅ Colonnes fixes (ID, statut, actions) — colonnes flexibles au milieu.
✅ Colonne Actions toujours en dernière position, alignée à droite.
❌ Ne jamais tronquer une colonne statut — toujours afficher le badge entier.

Side Navigation

Structure :

  • Logo Flow en haut (36px) + nom produit
  • Sections avec label uppercase (10px)
  • Items : icône 20px + label, height 36px
  • État actif : background cyan tint, texte cyan dark, font weight 500
  • État hover : background surface +1 niveau

Règles :

Largeur standard : 220px. Collapsible (icônes seules) à 52px.
Maximum 2 niveaux de navigation. Sous-items indentés de 28px.

Top Bar

Éléments gauche → droite :

  1. Logo/nom produit (fixe)
  2. Search bar globale (⌘K)
  3. Badge notifications (chiffre si > 0)
  4. Avatar utilisateur (initiales)

Modal

Types :

TypeUsageActions
ConfirmValider une action sensibleAnnuler + Confirmer
AlertInformer d'une erreur critiqueOK uniquement
FormSaisie dans un overlayAnnuler + Sauvegarder

Règles :

✅ Toujours un titre clair + corps explicatif + actions en bas à droite.
✅ Action destructive (Danger) toujours à droite, action annulation à gauche.
❌ Jamais plus de 2 actions principales dans une modal.
❌ Ne pas ouvrir une modal depuis une modal.

📏 Règles d'usage globales

Hiérarchie visuelle

  1. 1 seul point d'attention primaire par écran (CTA primary, alerte critique)
  2. Les données critiques (retards, alertes) sont toujours accompagnées d'une couleur sémantique
  3. Les KPIs utilisent la typographie grande (32px 300) — jamais de bold sur les grands chiffres

Dark Mode

  • Utiliser dark mode pour les tableaux de bord opérationnels utilisés en continu
  • Utiliser light mode pour les pages de configuration, formulaires, onboarding
  • Ne jamais mélanger les deux modes dans la même vue

Accessibilité

  • Contraste minimum texte/fond : 4.5:1 (WCAG AA)
  • Tous les champs de formulaire ont un label visible (pas de placeholder comme seul label)
  • Tous les boutons icône ont un tooltip ou aria-label
  • Focus ring visible sur tous les éléments interactifs
